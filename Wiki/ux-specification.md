# UX-спецификация

**Summary**: Минималистичный UX поверх технически богатого приложения. Один экран онбординга, главный экран в три элемента (таймер, кнопка, сервер), вся техническая сложность в «Расширенных».

**Sources**: VPN-клиент для macOS и iOS — Промт для Claude Code.md

**Last updated**: 2026-05-11

---

## Принцип

Primary-пользователь никогда не видит слов «Reality», «uTLS», «sniffing» на главном экране — только «Подключиться» и «Сменить локацию». Технически богатое приложение визуально минималистично.

Финальный дизайн — в Figma (параллельно с разработкой). Эта спека — поведение и структура.

## Онбординг

**Один экран. Первый запуск приложения.**

Структура:
- Логотип и название (центр-верх, ~25% высоты экрана)
- Очень короткий welcome-текст («Безопасный интернет в один тап» или подобный)
- Две кнопки внизу:
  1. **«Вставить из буфера обмена»** (primary) — проверяет буфер на `vless://`/`ss://`/`trojan://` или subscription URL. Импортирует или показывает сообщение «В буфере не найдена ссылка».
  2. **«Сканировать QR-код»** (secondary) — открывает камеру, при успехе импортирует.
- В углу — едва заметная кнопка «У меня уже есть конфиг файл» → file picker.

**Никаких приветственных слайдов про «что такое VPN»**, никаких 3-step tour'ов. После успешного импорта первого конфига онбординг больше никогда не показывается.

## Главный экран

Сверху вниз:

1. **Top bar**
   - Слева: иконка «Меню» (бургер) → sidebar (macOS) или modal-sheet (iOS) с разделами: Настройки, Расширенные, Помощь, О приложении
   - Центр: логотип
   - Справа: иконка «+» → sheet с тремя опциями: «Из буфера», «QR-код», «Из файла»
2. **Connection timer** — крупный шрифт, формат `HH:MM:SS`. При неактивном — `00:00:00`. Виден всегда.
3. **Большая центральная кнопка подключения** — круглая, ~30% высоты экрана.
   - `idle` — статичная, чёрная, серая иконка
   - `connecting` — анимация вращения
   - `connected` — акцентный цвет
   - `error` — чёрная, серая иконка
   - Тап в `connected` → отключение. Тап в `error` → принудительный retry.
4. **Статус подключения** — мелкий текст под кнопкой:
   - idle: «отключено»
   - connecting: «подключение»
   - connected: «подключено»
   - error: «ошибка подключения»
5. **Bottom bar** — кнопка выбора сервера → открывает список серверов
   - Текст: «Авто» или название текущего сервера
   - Иконка-индикатор справа: signal-strength по latency (зелёный/жёлтый/красный)

## Список серверов (v0.3 — реализовано)

Открывается как **sheet** при тапе на server-line главного экрана. Закрывается свайпом вниз или выбором сервера. Референс — Hiddify.

**Высота шита** — адаптивная (реализовано 2026-05-12):
- Если контент (заголовок + AutoCell + секции серверов) помещается в ≤88% высоты экрана → `presentationDetents([.height(estimatedHeight)])`. Шит точно по контенту, без лишнего пространства.
- Если контент больше → `presentationDetents([.large])`. Полноэкранный режим, контент скроллируется.
- Расчёт ведётся через статические константы в `ServerListSheet.swift` (serverRowH=80, autoCellH=116 и др.). **При применении Figma-макетов константы нужно пересмотреть** (фаза 11).

1. **Сверху** — ячейка «Авто» (auto-select по score = latencyMs × (1 + lossRate)). Отдельная секция без заголовка, всегда первая. Если выбрана — чекбокс. При тапе — режим «перед connect выбирать лучший», sheet закрывается.
2. **Секции по подпискам** — каждая секция с заголовком = имя подписки. Серверы без подписки (ручной paste-импорт) → секция «Добавлены вручную». Реализовано через `ScrollView + LazyVStack + Section` (не `List` — List несовместим с SwiftData FetchDescriptor + async latency updates).
3. **Строка сервера** — флаг страны (emoji из `cc=` URI-параметра или fragment regex, fallback 🌐) + имя + `LatencyBadge` (цветная капсула: 0-80 мс зелёный / 81-200 жёлтый / 201-500 оранжевый / 501+ красный / недоступен красный). Сервер с `isSupported=false` — плашка «не поддерживается», строка полупрозрачная. Название протокола скрыто.
4. **Pull-to-refresh** — два шага последовательно: (1) fetch всех subscription URL → merge новых серверов; (2) параллельный ping всех серверов. Один жест — полностью актуальный список. Автоматически при foreground (`.onChange(of: scenePhase)`).
5. **Удаление** — swipe по заголовку секции → «Удалить» → cascade delete (Subscription + все её ServerConfig). Swipe по строке сервера → удаление одного ServerConfig.
6. **Добавление подписки** — через существующую кнопку «+» на главном экране (TopBar). Subscription URL → создаёт `Subscription` @Model.
7. **Автореконнект при смене** — sheet закрывается → disconnect → ping → connect с новым сервером. Без алерта.
8. **Merge-стратегия** — при re-fetch подписки: новые URI добавляются, исчезнувшие помечаются `missingFromLastFetch=true` (не удаляются автоматически). `lastFetched` у `Subscription` обновляется.

> **Deferred (Phase 11):** поиск/фильтр в server list, редактирование имени подписки, фоновое обновление по расписанию (BGAppRefreshTask — Phase 6).

## Settings

Доступ через меню в Top bar главного экрана. Разделы:

- **Подписки** — список subscription URL, добавить/удалить/обновить
- **Уведомления** — placeholder на MVP, реальные push в roadmap (v1.6)
- **Внешний вид** — тема (system/light/dark), язык (auto/ru/en)
- **Безопасность** — Face ID/Touch ID toggle, тоггл [[kill-switch]] (вкл по дефолту)
- **Помощь** — FAQ, кнопка «Отправить лог разработчику», ссылка на TestFlight invite
- **О приложении** — версия, ссылка на open-source ядро (GitHub), лицензии
- **Расширенные** → отдельный экран

## Расширенные настройки

Скрытый раздел для tech-savvy пользователей:

- Ручной выбор протокола (override auto)
- DNS: выбор провайдера (Cloudflare/NextDNS/AdGuard/Quad9/custom), AdBlock toggle — см. [[dns-strategy]]
- Тоггл «Блокировать STUN» (WebRTC leak protection) — см. [[security-gaps]]
- Тоггл «Аналитика» (выкл аналитики)
- IPv6 mode (auto/tunnel/block) — см. [[ipv6-strategy]]
- uTLS fingerprint (random/Chrome/Firefox/Safari)
- Просмотр текущего [[rules-engine|rules.json]] (read-only)
- Кнопка «Принудительно обновить правила», версия rules.json
- Тоггл «Включить xray-core fallback»
- **macOS only**: тоггл «Отключить принудительную маршрутизацию» — выкл по умолчанию. Снижает детектируемость на macOS за счёт отключения `enforceRoutes`. Предупреждение: «может привести к утечке DNS» (см. [[security-gaps]] R5, формулировка черновая, уточнить в Figma)
- Конфиг-эдитор (только для текущей подписки) — для дебага
- Network diagnostics: ping monitor, traffic stats, latency graph

## Принципы UX

- Никаких баннеров, popup'ов про MAX-детект — всё тихо в локальный лог (см. [[max-messenger]])
- Биометрия — опциональна, выкл по дефолту
- Kill switch — **включён по дефолту**, тоггл для отключения
- Аналитика — включена по дефолту, тоггл для отключения (выкл аналитики)
