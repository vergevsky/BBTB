# Детект VPN приложениями

**Summary**: По данным анализа из источника, 22 из 30 популярных приложений в РФ детектируют наличие VPN на пользовательском устройстве. Из них **19 отправляют VPN-статус на сервер**. Это отдельный вектор угрозы — не сетевой DPI, а локальный детект.

**Sources**: Дыры в безопасности, которые нужно обсудить.md, ocr_methodika_vpn_proxy.md (раздел 6.3 — когда выполняется проверка)

**Last updated**: 2026-05-11

---

## Что это за угроза

Это **не** ТСПУ (см. [[tspu]] — сетевой DPI). Это **локальный** детект: приложение на устройстве пользователя проверяет, активен ли VPN, и:

1. может изменить своё поведение (например, отказать в работе сервиса)
2. может отправить статус на свой сервер — и оператор приложения / государство получает список пользователей с активным VPN

Это представляет угрозу для primary-аудитории проекта (см. [[product-overview]]) — у них на устройстве с большой вероятностью установлено хотя бы одно из этих приложений.

## Как они это делают

Эти приложения следуют [[rkn-methodology-document|официальной методике РКН]]. На наших платформах из неё применимо:

- iOS: `CFNetworkCopySystemProxySettings()`, `NWPathMonitor`, `NEVPNManager`, имена интерфейсов `utun*`
- macOS: `getifaddrs()`, анализ таблицы маршрутизации, `Transparent Proxy API`, факт активных Network Extensions

Конкретные API и оценка опасности — см. [[apple-detection-surface]].

На iOS sandbox ограничивает многие техники из методички (нет `dumpsys`, нет `/proc/*`, нет анализа маршрутизации), но `utun*`-интерфейсы обнаружимы.

## Когда они это проверяют

Согласно методике (раздел 6.3), приложения проверяют VPN **в момент**:
- запуска приложения
- аутентификации
- выполнения ключевого действия (подтверждение покупки/перевода, указание конечной точки маршрута)

**Не непрерывно** — методика прямо запрещает это (экономия трафика и батареи).

Это объясняет паттерн: банки проверяют VPN на login и при оплате; такси — при указании маршрута; маркетплейсы — при оформлении заказа.

## Что это значит для проекта

- Наш VPN-клиент **не маскирует** факт активного VPN на уровне iOS — приложения смогут это детектировать
- Это **открытый вопрос для обсуждения** — см. [[security-gaps]]
- Для пользователей, которым критично использовать конкретные банковские/госуслуги приложения — нужен механизм `never_through_vpn` через [[rules-engine|rules.json]] (трафик идёт мимо туннеля, приложение видит обычную сеть)
