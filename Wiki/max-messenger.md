# MAX (мессенджер)

**Summary**: Российский государственный мессенджер MAX рассматривается в проекте как сущность, требующая детектирования (без UI-уведомлений) и блокировки через [[rules-engine|rules.json]]. Также входит в список приложений, [[vpn-detection-by-apps|детектирующих VPN]].

**Sources**: VPN-клиент для macOS и iOS — Промт для Claude Code.md, Дыры в безопасности, которые нужно обсудить.md

**Last updated**: 2026-05-11

---

## Контекст

MAX — российский мессенджер, продвигаемый как государственная альтернатива WhatsApp/Telegram. Известные домены: `max.ru`, `mssgr.tatar.ru`. В контексте VPN-проекта:

- является одним из приложений, которые детектируют VPN на устройстве (см. [[vpn-detection-by-apps]] — MAX в списке 22 из 30)
- известные домены MAX добавляются в `block_completely` через [[rules-engine|rules.json]]

## Детектирование на устройстве

Цель — узнать, установлен ли MAX на устройстве пользователя, для адаптации `rules.json` и отладки. **Без UI-уведомлений в обычном flow.**

**iOS:**
- `UIApplication.canOpenURL(URL(string: "max://")!)`
- URL-схему добавляем в `LSApplicationQueriesSchemes` в `Info.plist`
- Если схема не известна на момент разработки — выясняем актуальную и обновляем

**macOS:**
- `NSWorkspace.shared.urlForApplication(withBundleIdentifier:)` (если bundle id известен)

## Что делаем с информацией

- **БЕЗ UI-уведомлений в обычном пользовательском flow.** Никаких баннеров, popup'ов, экранов в onboarding.
- Информация записывается **в локальный debug-лог**.
- Когда пользователь нажимает «Отправить лог разработчику» — администратор видит у этого пользователя факт установки MAX.
- Администратор может на основе этой телеметрии адаптировать `rules.json` — например, добавить специфичные паттерны в block-list.

## Блокировка через rules.json

Известные домены MAX добавляются в секцию `block_completely` — соединения дропаются вне зависимости от состояния VPN:

```json
"block_completely": {
  "domains": ["max.ru", "mssgr.tatar.ru"]
}
```

## App Store risk

Функция детекта использует только публичный API (`canOpenURL:`) — риски ревью минимальные.

Описание в App Privacy: «приложение проверяет наличие конфликтующих VPN-приложений для совместимости».
