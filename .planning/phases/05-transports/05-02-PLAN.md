<objective>
Wave 1 — WebSocket vertical slice. Этот wave **меняет публичный API** парсеров и моделей:
`ParsedVLESSTLS.networkType: String` удаляется, `ParsedTrojan.TransportType` enum удаляется,
оба заменяются на `transport: TransportConfig`. Парсеры VLESS+TLS и Trojan теперь делегируют
парсинг transport query-params на `TransportParamParser`. Регистрируется первый "настоящий"
transport handler — `WSTransportHandler`, который produce sing-box `transport: {type: "ws", ...}`
JSON блок.

После Wave 1: WebSocket URI парсится → ParsedXxx.transport == .ws(path:host:) → может быть
передан в TransportRegistry для построения JSON блока. PoolBuilder + protocol package
buildOutbound ещё не используют TransportRegistry — это придёт в Wave 5. Wave 1 — изолированный
шаг миграции данных без интеграции с PoolBuilder.

Purpose: Изолировать самый сложный refactor (миграция типов в моделях ParsedXxx) в один wave.
WebSocket выбран первым потому что у Trojan уже есть `TransportType.ws` (D-06 mapping
1-to-1: `case ws(path, host)` → `case ws(path, host)` без потери семантики). Test fixture
`trojan-ws-user-fixture.txt` (Phase 2) — единственный backward-compat invariant; должен
продолжить парситься без изменения внешней семантики.

Output: 5 модифицированных existing файлов (ImportedServer, TrojanURIParser, VLESSURIParser, 2 test
файла) + 2 новых файла (WSTransportHandler.swift + tests) + 2 fixture файла.
</objective>
