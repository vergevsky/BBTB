не содержит `host` query param — массив пустой (sing-box использует TLS `server_name` как
authority по умолчанию). HTTPUpgrade (Wave 3) и WS (Wave 1) — host string, не array.

handler-е, покрыть тестами все edge cases (host empty / single-element / multi-element future-proof),
и добавить URI fixtures для VLESS+TLS+HTTP и Trojan+HTTP.

Output: 1 новый handler + 1 test файл + 2 URI fixture + дополнения в 2 существующих test файла.

HTTPTransportHandler signature:
```swift
public enum HTTPTransportHandler: TransportHandler {
    public static let identifier = "http"
    public static let displayName = "HTTP/2"
    public static let supportedProtocols = ["vless-tls", "trojan"]
    public static func buildTransportBlock(for config: TransportConfig) -> [String: Any]?
}
```

sing-box HTTP transport JSON shape (Pitfall 7, Example 2 lines 728-740):
```
.http(path: "/api") → ["type": "http", "path": "/api"]
```

The host field is COMPLEX: TransportConfig.http carries only `path`, not host. The host comes
from the protocol package's tls.server_name (SNI). In Wave 2 we do NOT introduce host into
HTTPTransportHandler — the handler emits only `{type, path}`. The protocol package's
buildOutbound (Wave 5) decides whether to inject `"host": [sni]` based on TLS context.

This intentional design choice matches sing-box semantics: if HTTP transport's `host` is
omitted, sing-box falls back to tls.server_name. Our minimal handler output is safe.

URI query → TransportConfig.http mapping (TransportParamParser handles in Wave 0):
- `?type=http&path=/api` → .http(path: "/api")
- `?type=h2&path=/api` → .http(path: "/api") (h2 alias)
- `?type=http` (no path) → throws .httpMissingPath
