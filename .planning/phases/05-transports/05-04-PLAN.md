sing-box HTTPUpgrade JSON shape (verbatim from RESEARCH Example 2 lines 745-752):
```
.httpUpgrade(path: "/upgrade", host: "example.com") вЖТ 
  ["type": "httpupgrade", "path": "/upgrade", "host": "example.com"]
```

**Critical difference from HTTP transport:** `host` is STRING, not array. This matches sing-box
schema strictly (Pitfall 7). If we emit array here, sing-box validator rejects with "expected
string for host".

If `host` is empty (URI provided no `?host=` param) вАФ omit the `host` key entirely. sing-box
will fall back to internal default (TLS authority).

URI mapping (already in TransportParamParser, Wave 0):
- `?type=httpupgrade&path=/u&host=h` вЖТ .httpUpgrade(path: "/u", host: "h")
- `?type=httpupgrade&path=/u` вЖТ .httpUpgrade(path: "/u", host: "")
- `?type=httpupgrade` (no path) вЖТ throws .httpUpgradeMissingPath
