sing-box gRPC JSON shape (verbatim from RESEARCH Example 2 lines 720-724):
```
.grpc(serviceName: "tunsvc") → ["type": "grpc", "service_name": "tunsvc"]
```

**Critical: JSON field is `service_name` (snake_case), URI param is `serviceName` (camelCase) — Pitfall 6.**

URI mapping (TransportParamParser, Wave 0):
- `?type=grpc&serviceName=tunsvc` → .grpc(serviceName: "tunsvc")
- `?type=grpc` (no serviceName) → .grpc(serviceName: "TunService") — sing-box default per Open Question 5
- `?type=grpc&serviceName=` (empty value) → .grpc(serviceName: "") — empty allowed, sing-box decides

`mode` and `authority` URI params — IGNORED in Phase 5 (defaults applied by sing-box).
