## Phase Goal

**As a** user of BBTB with multiple servers in my subscription, **I want to** have the app automatically try the next server after my current server has failed 3 reconnect attempts, **so that** transient or persistent failures on one exit don't take my VPN down for the entire session.

<objective>
Wave 6 (final integration wave) — Failover. Replaces the `NoFailoverProvider` stub from Wave 5 with `SwiftDataFailoverProvider` that owns a cursor into the supported ServerConfig list and advances on each call. Wires in the cursor-reset triggers (manual disconnect + 30s+ stable session per Pitfall 4) and the edge-case notifications (single-server + all-exhausted).

This wave is `autonomous: false` because the final acceptance involves device-side UAT (forcing a server failure on an iPhone with multiple subscription servers and watching the failover banner). The implementation itself is autonomous; the UAT checkpoint at the end is human-verify.

Purpose: NET-11 (failover on server failure). Closes the Phase 6 NET-* group.

Output:

- `TunnelController.swift` extended with `scheduleFailoverResetAfterStableSession` + reset on manual disconnect.
- `MainScreenViewModel.swift` injection wiring (replace NoFailoverProvider with SwiftDataFailoverProvider).
- Tests covering round-robin advance, reset conditions, edge cases.
- Human-verify checkpoint for device UAT.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-W6-01 | DoS | Reset triggered prematurely → infinite re-try on same broken server | mitigate | Task 2 stable-session timer uses 30s threshold AND startedAt-guard pattern: if status leaves .connected before 30s, the timer task observes the change and skips reset. Test 10 verifies. (Pitfall 4.) |
| T-06-W6-02 | DoS | All-servers exhausted but stateMachine keeps trying | mitigate | FailoverProvider.nextServerAttempt returns nil after full cycle; ReconnectStateMachine transitions to .allFailed and exits its loop. Test 3 verifies. |
