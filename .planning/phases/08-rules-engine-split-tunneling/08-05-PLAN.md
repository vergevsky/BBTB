<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| iOS BGTaskScheduler → app process | OS controls when handler fires; app must register identifier during init |
| App Group entitlement → BGAppRefreshTask handler | handler runs in same process so has same entitlements |
| Scene phase observation → coordinator state | scenePhase callbacks run async; @MainActor isolation для UI |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-W4-01 | DoS | User отключил Background App Refresh в iOS Settings | mitigate | Foreground sanity fetch (12h threshold) бэкап путь — invocation на каждом app open closes the gap if BG-task disabled (Pitfall 2) |
| T-08-W4-02 | DoS | BGAppRefreshTask never reschedules — gets stuck forever | mitigate | handleRefreshTask `setTaskCompleted` called regardless of success; new submit запрашивается также regardless (try? swallows errors). Pitfall 2 mitigation. |
| T-08-W4-03 | Tampering | Wrong identifier mismatch BGTaskScheduler.register vs Info.plist | mitigate | Identifier hardcoded в obeих местах как const string `"app.bbtb.client.ios.rules-refresh"`; W7 invariant check `grep` confirms match |
| T-08-W4-04 | DoS | rulesCoordinator.bootstrap() blocks main thread на cold-start | mitigate | `Task.detached(priority: .utility)` — bootstrap runs on background queue; cold-start defer per DEC-06d-01 |
| T-08-W4-05 | Information disclosure | BGAppRefreshTask handler logs sensitive data | accept | Coordinator logs only operation type + counts + mirror URL (public-by-design); no PII / no actual rules content в logs |
| T-08-W4-06 | Repudiation | Foreground sanity fetch silent failure | mitigate | RulesEngineLogger.coordinator уже logs each fetch attempt + failure reason (W2); persistent inline status в UI surfaces к user через next force-update view |
| T-08-W4-07 | Elevation of privilege | NSBackgroundActivityScheduler runs с full app sandbox permissions | accept | No new permissions vs main app process; App Group entitlement preserved; macOS NSBgActivity не требует additional entitlement (RESEARCH § Pattern 4) |
| T-08-W4-08 | DoS | UIBackgroundModes 'fetch' enables iOS-wide app refresh budget consumption | accept | Budget shared across apps; iOS power-aware schedules opportunistically; 30s per launch upper bound |
</threat_model>
