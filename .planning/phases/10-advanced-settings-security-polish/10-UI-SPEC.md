# Phase 10 — UI Design Contract

> Visual and interaction contract for Phase 10 (Advanced Settings + Security polish).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Design standard:** Apple Human Interface Guidelines (HIG) for iOS 18 / macOS 15. SwiftUI
> native components only. System fonts (SF Pro). Semantic system colors only (no hardcoded
> hex). All design tokens — Apple platform defaults; no custom design system. Dark Mode
> handled automatically via semantic colors. Russian primary language (LOC-01).
>
> **Why no shadcn / no custom tokens:** BBTB — native SwiftUI app для iPhone/iPad/Mac. shadcn
> is a React/web library and is not applicable. Apple HIG is the authoritative design system
> for this project. All "spacing scale" / "typography scale" / "color tokens" rows below map
> to Apple's published platform defaults rather than to project-local constants.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (Apple HIG native — SwiftUI primitives) |
| Preset | not applicable (no shadcn — this is a native iOS/macOS app) |
| Component library | SwiftUI `Form` / `Section` / `Toggle` / `Picker` / `Stepper` (iOS 18+, macOS 15+) |
| Icon library | Apple SF Symbols 6 (no custom icons; use only `Image(systemName:)`) |
| Font | SF Pro (system font) via `.font(.body)` / `.font(.headline)` / `.font(.footnote)` |
| Color system | Semantic system colors (`.primary` / `.secondary` / `.accentColor` / `Color(.systemBackground)` / `Color(.secondarySystemGroupedBackground)` / `.red` semantic) — Dark Mode automatic |
| Material | `Color(.systemGroupedBackground)` for Form base on iOS; `Form` style auto on macOS |
| Touch targets | ≥ 44 pt (Apple HIG minimum) — `Toggle` / `Picker` rows in `Form` are 44 pt by default |
| Haptics | Light impact (`UIImpactFeedbackGenerator(style: .light)`) on toggle changes — only on STUN-block enable confirmation alert dismiss (see §Interaction) |

**Rationale (from CLAUDE.md):** проект native Apple, без сторонних UI-стилевых систем. Каждый
toggle следует SwiftUI defaults — это даёт автоматический Dark Mode, Dynamic Type,
VoiceOver и поддержку platform-version вариаций без custom-кода.

## Spacing Scale

Apple HIG defaults — no project-specific scale.

| Token | Value | Usage |
|-------|-------|-------|
| Form row vertical | 11 pt (system) | Default `Form` row vertical padding (iOS) |
| Form row horizontal | 16 pt | Default `Form` content horizontal inset (iOS) |
| Section spacing | 35 pt (system) | Spacing between Form sections (iOS grouped) |
| Header bottom gap | 6 pt | Section header to first row (system) |
| Footer top gap | 6 pt | Last row to section footer (system) |
| Touch target min | 44 pt × 44 pt | All `Toggle`, `NavigationLink`, `Button` rows |
| Icon ↔ label inline gap | 8 pt | When a row uses `Label("…", systemImage: "…")` |

**Exceptions:** none. All spacing comes from SwiftUI `Form` system defaults. **Do NOT**
introduce `.padding(N)` modifiers on rows — they break HIG grouping. If a row needs an
explanatory note, use a Section `footer:` (system-spaced), not in-line padding.

## Typography

Apple HIG text styles (`.font(.body)` etc.). No fixed pixel sizes — Dynamic Type honored
throughout.

| Role | Text style | Weight | Usage in Phase 10 |
|------|-----------|--------|-------------------|
| Section header | `.font(.footnote)` + `.foregroundStyle(.secondary)` + `.textCase(.uppercase)` (iOS default) | regular (400) | "DNS", "ANTI-DPI", "БЕЗОПАСНОСТЬ", "ПРАВИЛА" — section labels |
| Row label | `.font(.body)` | regular (400) | All Toggle / Picker labels (e.g. "CDN-фронтинг", "Cert pinning") |
| Row value (Picker) | `.font(.body)` + `.foregroundStyle(.secondary)` | regular (400) | Selected uTLS fingerprint name (`.menu` style trailing value) |
| Footer warning | `.font(.footnote)` + `.foregroundStyle(.secondary)` | regular (400) | "Сломает видеозвонки в браузере…" |
| Footer destructive | `.font(.footnote)` + `.foregroundStyle(.red)` | regular (400) | macOS enforceRoutes OFF state: "Выкл = трафик идёт напрямую если VPN упал…" — color flips to `.red` while OFF |
| Inline status (cooldown / error) | `.font(.footnote)` + semantic color | regular (400) | Reuses Phase 8 `ForceUpdateRulesButton` pattern |
| Picker menu items | `.font(.body)` (system default) | regular (400) | "Случайный" / "Chrome" / "Firefox" / … in `.menu` Picker |

**Hard rule for Phase 10:** **никогда не задавать explicit font size в `pt`**. Используем
только text-style constants (`.body`, `.footnote`, `.headline`, `.subheadline`, `.caption`)
— это даёт Dynamic Type поддержку. Это покрывает Apple HIG → User Settings → Larger Text.

## Color

Semantic system colors only. No `Color(hex: 0xFFAA00)` — every color must be `Color(.system…)`
or a semantic accent (`.primary`, `.secondary`, `.accentColor`, `.red`, `.orange`).

| Role | SwiftUI value | Usage |
|------|---------------|-------|
| Dominant (60%) — Form background | `Color(.systemGroupedBackground)` (iOS) / Form auto (macOS) | Behind all sections |
| Secondary (30%) — Row background | `Color(.secondarySystemGroupedBackground)` (iOS) | Default `Section` row container |
| Accent (10%) | `.accentColor` (system tint — синий по умолчанию на iOS, configurable в Asset Catalog) | Toggle ON state, Picker selected highlight, NavigationLink chevron |
| Primary text | `.primary` | All row labels |
| Secondary text | `.secondary` | Footers, picker trailing value, disabled state |
| Destructive | `.red` (semantic, auto Light/Dark) | macOS enforceRoutes OFF footer; `Confirm` STUN-block destructive button in alert |
| Warning highlight | `.orange` | Reused Phase 8 `MinAppVersionBanner` row tint (`Color.orange.opacity(0.15)` background) — already in code, unchanged |

**Accent reserved for (explicit list — нельзя расширять без UI-checker review):**

1. Toggle ON state thumb/track (system).
2. Selected `Picker` row in `.menu` / `.navigationLink` style (system).
3. `NavigationLink` disclosure chevron (system).
4. Inline link in footer text (`Text("…подробнее")` with `.foregroundStyle(.tint)`) — **not used in Phase 10** but reserved.

**Forbidden in Phase 10:**

- Hardcoded hex literals (`Color(red: …)`, `Color(hex:)`).
- `.background(.blue)` / `.background(.green)` etc. (must be `.accentColor` or `Color(.…)`).
- Custom traffic-light dots: STUN-block warning row uses footer text in `.red`, not a colored circle.
- Tinting Section row backgrounds for non-warning purposes (Phase 8 orange banner is the only exception).

### Copywriting rules for Phase 10

1. **Toggle labels are nouns/noun-phrases**, not commands. "CDN-фронтинг" ✓, "Включить CDN" ✗.
2. **Footer copy explains *consequence*, not the underlying technology**. "Сломает видеозвонки в браузере" ✓, "Блокирует UDP-порты 3478 и 5349" ✗.
3. **Russian primary, English duplicate**. No mixed-language strings ("CDN fronting" в русском UI ✗).
4. **No exclamation marks, no emoji** in any string (Apple HIG: avoid emotional punctuation in system surfaces).
5. **Destructive action** uses verb-first imperative ("Включить" / "Enable"). Cancel uses "Отмена" / "Cancel".
6. **Empty state** — нет в Phase 10 (Advanced Settings всегда имеет содержимое: либо toggles, либо MinAppVersion banner, либо rules viewer).
7. **Error state** — переиспользуется Phase 8 `forceUpdateStatusOutcome` inline status pattern для cert-pin-manifest fetch errors (если planner добавит UI для pin manifest debug — discretion). Otherwise pin failures surface as banner на main subscription-refresh path (existing).

```
      // ── 5. Rules Section (Phase 8, existing) ───────────────────────
      Section { RulesViewerSection(snapshot: viewModel.rulesSnapshot) }

        ForceUpdateRulesButton(…)

```

**Section ordering rationale:**

1. **MinAppVersionBanner first** — critical safety banner (Phase 8 UI-SPEC §A-08, preserved).
2. **DNS before Anti-DPI** — DNS is most-frequently changed (Phase 6 UAT showed это).
3. **Anti-DPI before Безопасность** — Anti-DPI содержит 4 toggle (visible weight); Безопасность — 1-2 (compact).
4. **Rules last** — viewer + force-update — rarely accessed, OK at the bottom.

**Per-section content ordering inside Anti-DPI (D-15 discretion, locked here):**

1. CDN-фронтинг (toggle) — глобальная стратегия (most-impactful).
2. Mux (toggle) — secondary anti-DPI knob.
3. uTLS отпечаток (picker — `.menu` style) — visual differentiation from toggles.
4. Блокировать STUN (toggle) — sensitive, ставится последним чтобы пользователь прочитал
   предыдущие footers (по сути расширенный context до warning).

**Per-section content ordering inside Безопасность:**

1. Cert pinning (toggle, cross-platform).
2. Принудительная маршрутизация (toggle, **macOS only**, `#if os(macOS)`).

## Interaction Contract

### Toggle interactions (standard)

| Action | Behavior |
|--------|----------|
| Tap toggle | SwiftUI standard animation (system); `@AppStorage` writes synchronously |
| Light haptic | None for plain toggles (system handles); haptic only on STUN confirmation alert dismiss |
| Persistence | `@AppStorage` (UserDefaults Suite — App Group) — survives kill |
| Apply to running tunnel | **Не применяется live.** Banner-style notice **не показываем в Phase 10** (Phase 6 KILL-03 pattern уже задаёт «применяется при следующем connect»; consistent across all toggles) |
| Live-apply hook | Mux / STUN-block / CDN — на next connect. Cert pinning — on next subscription refresh. macOS enforceRoutes — on next `applyCurrentStateToCachedManager()` (already wired via Phase 6c) |

### STUN-block toggle — destructive confirmation (D-16)

When user flips STUN-block toggle from OFF → ON:

1. Toggle visually animates ON (system).
2. **Immediately** present `.alert(…)` with:
   - Title: `settings.antiDpi.stun.confirm.title` → "Включить блокировку STUN?"
   - Message: `settings.antiDpi.stun.confirm.message` → "Видеозвонки в браузере перестанут работать…"
   - Buttons:
     - Primary: `settings.antiDpi.stun.confirm.action` ("Включить") — `.destructive` role, **keeps toggle ON**, triggers light haptic on dismiss.
     - Cancel: `settings.antiDpi.stun.confirm.cancel` ("Отмена") — `.cancel` role, **reverts toggle to OFF**.
3. On OFF → ON: no confirmation (back to safe default).

Implementation pattern (recommended for planner):

```swift
@AppStorage("app.bbtb.stunBlockEnabled") var stunBlockEnabled: Bool = false
@State private var pendingStunEnable: Bool = false  // local UI-only
// Wrap Toggle binding so OFF→ON gates through the alert.
```

### uTLS fingerprint picker — pattern

- **Style:** `Picker(…, selection: $utlsFingerprint) { … }.pickerStyle(.menu)` on both iOS and macOS.
- **Why `.menu` not `.navigationLink`:** 7 options is small enough to fit a popover menu; saves a navigation hop. (`.navigationLink` reserved for future expansion if list grows past 12.)
- **Selected value display:** trailing `.secondary`-colored value inside the menu trigger (system default).
- **Default value:** `"random"` (D-09 / Phase 7a smart default preserved). Default label: "Случайный".
- **Persistence:** stored as `String` key in `@AppStorage`. Values are sing-box fingerprint identifiers (`"random"`, `"chrome"`, `"firefox"`, `"safari"`, `"ios"`, `"android"`, `"edge"`) — **never localized at storage layer** (Russian/English label is presentation-only).

### macOS enforceRoutes toggle — destructive footer color flip

- **Default:** ON (corresponds to `enforceRoutes=true`, current production behavior).
- **OFF state footer color flips to `.red`** to signal degraded security posture (HIG: warning color for non-blocking risk).
- **Toggle key naming:** UI-level `macOSEnforceRoutesEnabled` (positive sense — user-visible) maps to internal `macOSDisableEnforceRoutes` (negative sense — implementation flag). Planner handles the inversion.
- **iOS:** `EnforceRoutesToggleRow` is gated `#if os(macOS)` — iOS does not render this row (iOS 26 forces `includeAllNetworks`; toggle would mislead).

### CDN / Mux / Cert pinning toggles — plain toggles

- No confirmation alerts (D-08 / D-14 / D-07 — all are reversible runtime overrides).
- CDN: if `cdnFrontingEnabled` is ON but no server has `frontingProfile` blob in subscription → no UI warning (settings page reflects intent; effect resolved at connect time). Document in footer text.

## Accessibility Contract (VoiceOver + Dynamic Type)

All new rows MUST provide:

1. **`accessibilityLabel(_:)`** — speakable form of the row purpose, distinct from visual label.
2. **`accessibilityHint(_:)`** — what happens when activated ("Дважды коснитесь чтобы включить блокировку STUN").
3. **`accessibilityValue(_:)`** for Picker — currently selected option name.
4. **`accessibilityIdentifier(_:)`** for UI testing — stable identifier, English, kebab-case-like (matches existing pattern `bbtb.settings.adblock.toggle`).

**Why STUN-block lives on both platforms:** WebRTC leak is platform-agnostic; the sing-box
`route.rule` injection works identically on iOS and macOS extensions.

`includeAllNetworks=true` and ignores the `enforceRoutes` field. Showing a non-functional
toggle on iOS would be misleading (Apple HIG: never show controls that do nothing).
