В `SingBoxConfigLoader.swift` добавить **новый приватный helper** перед `expandConfigForTunnel`:

```
/// DPI-05 / Phase 10 — protocol whitelist для Mux injection (D-09).
/// Возвращает true только для VLESS+TLS plain, Trojan, Shadowsocks (включая 2022-blake3-* AEAD).
/// Возвращает false для VLESS+Reality, VLESS+Vision, TUIC, Hysteria2 и всех остальных типов.
private static func isMuxCompatible(_ outbound: [String: Any]) -> Bool
```

Внутренняя логика (без code block в action; описываю поведение):
- Достать `type` как String. Если nil → return false.
- Если `type == "trojan"` → return true.
- Если `type == "shadowsocks"` → return true.
- Если `type == "vless"`:
  - Если `outbound["flow"] as? String` содержит подстроку `"xtls-rprx-vision"` (case-sensitive — точное sing-box значение) → return false.
  - Если `outbound["reality"] as? [String: Any]` присутствует (любое значение) → return false. (Старая 1.9 схема — отдельный ключ reality на верхнем уровне outbound.)
  - Если `outbound["tls"] as? [String: Any]` существует И в нём `["reality"] as? [String: Any]` существует И `["reality"]["enabled"] as? Bool == true` → return false. (Sing-box 1.10+ схема — reality внутри tls.)
  - Иначе → return true (VLESS+TLS plain).
- Любой другой type → return false.
