<objective>
Wave 2 — реализуем **DPI-08 certificate pinning** для subscription URL endpoint (Marzban). Минимальный D-13 scope: пиннинг только в main app, только для SubscriptionURLFetcher (rules.json уже Ed25519-protected).

Архитектура (D-11..D-14):
1. `PinnedSessionDelegate` — единственный класс, NSObject + URLSessionDelegate; SPKI SHA-256 matcher через Apple-standard `SecTrustCopyCertificateChain` + `SecCertificateCopyKey` + `SecKeyCopyExternalRepresentation` + `CryptoKit.SHA256`.
2. `PinStore` — bootstrap pins (hardcoded байты в PinStore.swift) + merged manifest pins (от SubscriptionPinManager).
3. `SubscriptionPinManager` actor — bootstrap (bundle → App Group) + performBackgroundRefresh (HTTPS+Ed25519 verify, идентичный pattern RulesEngineCoordinator); validUntil hard reject (рекомендация RESEARCH.md).
4. `PinnedSubscriptionURLFetcher` — SubscriptionURLFetching struct, оборачивает SubscriptionURLFetcher.fetch с PinnedSessionDelegate.
5. `scripts/generate-spki-pin.swift` — CLI helper для генерации pins в форматe, совместимом с client-side `SecKeyCopyExternalRepresentation` (verifies A4 assumption из RESEARCH.md — критично).

Purpose: защитить trust path к Marzban от MITM (rogue CA, captive portal injection). После Wave 2 пользователь может (в Расширенных) включить certPinningEnabled toggle (default ON в Wave 1) и subscription refresh откажется работать если сертификат VPS подменён.
