# Phase 11 — Figma Design Spec

Что нужно отрисовать для Phase 11. Новые экраны помечены **[NEW]**, доработка существующих — **[POLISH]**, новые элементы внутри существующих экранов — **[NEW ELEMENT]**.

---

## 1. Onboarding Screen **[NEW]**

Показывается только при первом запуске (пока серверов нет).

| Элемент | Описание |
|---------|----------|
| Title | Главный заголовок (например: «Верни жука» / «Добро пожаловать») |
| Subtitle | Одна строка — суть приложения без объяснения что такое VPN |
| CTA #1 — Primary | Кнопка «Вставить из буфера» (большая, accent color) |
| CTA #2 — Secondary | Кнопка «Сканировать QR» (меньше или outline-стиль) |
| Фон / общий лэйаут | Центрированный контент, без Navigation Bar |

**Нужно 1 вариант:** светлая тема + тёмная тема (если DesignSystem поддерживает).

## 2. Connection Button — все состояния **[POLISH]**

Кнопка уже есть в коде (`ConnectionButton.swift`). Нужно уточнить внешний вид каждого состояния.

| Состояние | Текущий цвет | Что нужно уточнить |
|-----------|-------------|-------------------|
| `empty` | Серый (disabled) | Без изменений? |
| `idle` | Серый (enabled) | Без изменений? |
| `connecting` | Оранжевый | **Спиннер** — где, какой тип, размер, поверх или вместо иконки |
| `connected` | Accent (синий/зелёный) | Без изменений? |
| `error` | Красный | Без изменений? |

**Ключевой вопрос:** спиннер при `connecting` — это:
- отдельное кольцо вокруг кнопки?
- иконка внутри кнопки меняется на спиннер?
- пульсирующее кольцо?

## 3. Main Screen **[POLISH]**

Существующий экран, нужна Figma-версия для pixel-perfect реализации.

### 3a. Idle / Empty состояние (нет серверов)
| Элемент | Описание |
|---------|----------|
| Top Bar | ≡ (Settings) слева, + (Import) справа |
| Центральная пустая карточка | Две кнопки: «Вставить из буфера» + «Сканировать QR» |

### 3b. Idle состояние (серверы есть)
| Элемент | Описание |
|---------|----------|
| Top Bar | ≡ слева, + справа |
| Status Pill | Текст «Отключено» (или пусто) |
| Connection Button | Серый, enabled |
| Server Line | Название сервера + иконка |
| Timer | «00:00:00» (скрыт или серый) |

### 3c. Connecting состояние
| Элемент | Описание |
|---------|----------|
| Status Pill | «Подключение…» |
| Connection Button | Оранжевый + **спиннер** (см. п.2) |
| Server Line | Без изменений |

### 3d. Connected состояние
| Элемент | Описание |
|---------|----------|
| Status Pill | «Подключено» |
| Connection Button | Accent color |
| Timer | Активный таймер (например «00:02:15») |
| Server Line | Активный сервер |

## 4. Server List Sheet **[POLISH]**

В коде есть статические константы высот. Нужны точные значения из Figma.

| Элемент | Текущая высота в коде | Нужно в Figma |
|---------|-----------------------|---------------|
| Server Row | `serverRowH = 80` | Точная высота строки сервера |
| Auto-select Cell | `autoCellH = 116` | Точная высота ячейки авто-выбора |
| Subscription Header | `subHeaderH = 44` | Точная высота заголовка подписки |
| Manual Section Header | `manHeaderH = 36` | Точная высота заголовка ручных серверов |
| Empty Card | `emptyCardH = 220` | Карточка пустого состояния |

**Почему важно:** эти значения влияют на высоту шита (`presentationDetents`). Если значения в коде не совпадут с Figma — шит будет открываться на неправильной высоте.

## 5. FAQ / Help **[NEW]**

**Размещение (решено):** Кнопка «Помощь» в Settings → NavigationLink → отдельный `HelpView`.

**Структура FAQ-экрана (контент):**
- «Как добавить сервер?» — краткий ответ
- «Что делать если не подключается?»
- «WebRTC leak — что это и как влияет?»
- «Приложения которые всё равно видят VPN» — 22 приложения из РФ (DETECT-03)
- «Известные ограничения детектирования»

---

## 6. Кнопка «Отправить лог разработчику» **[NEW ELEMENT]**

**Размещение (решено):** Секция «Диагностика» в Settings — отдельный `Section` под всеми остальными.

| Элемент | Описание |
|---------|----------|
| Кнопка | «Отправить лог разработчику» — нейтральный стиль (не деструктивный) |
| Footer под кнопкой | «Последние 24ч. IP-адреса маскируются.» |
| Под footer | Версия приложения + версия ОС (мелким шрифтом) |
| Состояние загрузки | Спиннер / `ProgressView` пока идёт сбор |
| Результат | Share Sheet открывается сразу после сбора |
