## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OS LaunchServices → BBTB.app via `bbtb://` | Custom-scheme URL is system-delivered to `.onOpenURL`. URL data is trusted-format (URL type) но содержимое — externally controlled. |
| OS NSUserActivity → BBTB.app via Universal Link | Apple CDN verifies AASA before delivery; reaches us only for domains matching `applinks:import.bbtb.app`. Stronger guarantee than custom scheme. |
| Entitlements file → Apple build chain | Static XML — no runtime parsing in our code. Trust delegated to xcode/codesign. |
| Info.plist CFBundleURLTypes | Static declaration; declares scheme but does not implement handler. Implementation lives в SwiftUI modifier. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Spoofing | bbtb:// custom scheme registration | accept | Other apps могут register same scheme (Pitfall #6). Mitigation: Universal Link primary path; custom scheme secondary. FAQ documents limitation. |
| T-09-02 | Tampering | URL passed through to MainScreenViewModel.handleDeepLink | mitigate | URL passes through DeepLinkRouter → ImportHandler validation (Plan 02 Task 2.2 covers пidstuff#5 double-encode etc.). VM just shuttles error.localizedDescription to UI alert. |
| T-09-03 | Repudiation | DeepLinksLogger router/importer категории | accept | Notice-level logs за `privacy: .public` show only scheme/host (Foundation default for URL string privacy). Console.app for debug. No persisted audit log (matches existing project posture; v0.9 audit log deferred per Phase 2 T-02-03). |
| T-09-04 | Information Disclosure | Apple App Site Association (AASA) hosting (Wave 4) | mitigate | HTTPS-only fetch; Apple CDN validates cert chain. Our AASA contains только App IDs + path components — no secrets. |
| T-09-05 | DoS | Cold-start race (pendingDeepLink buffer) | mitigate | Bounded: single URL slot (@State var). If user spams таппings while app is launching, only the LAST URL is buffered (subsequent overwrites previous). This is acceptable per UX-SPEC F1 — user sees only one import attempt on cold-start; subsequent taps в warm-start state fire immediately. |
| T-09-06 | EoP | DeepLinkRouter.register from App.init Task.detached | accept | Task.detached priority: .utility — non-blocking.
