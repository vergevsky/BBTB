    # Phase 9 — AASA Deploy Runbook

    > **Цель:** Опубликовать `apple-app-site-association` файл на `https://import.bbtb.app/.well-known/apple-app-site-association`.
    >
    > **Источники:** `09-RESEARCH.md` § Pattern 6 + Pitfall 2 + Example 7; `09-CONTEXT.md` D-01 + D-02.

    ## AASA Content (точное содержимое)

    ```json
    {
      "applinks": {
        "details": [{
          "appIDs": [
            "UAN8W9Q82U.app.bbtb.client.ios",
            "UAN8W9Q82U.app.bbtb.client.macos"
          ],
          "components": [{ "/": "/import*" }]
        }]
      }
    }
    ```

    **Не добавлять:** EOL после последнего `}`; trailing whitespace; UTF-8 BOM. Файл должен быть ASCII / минимальный UTF-8.

    ## Option A — nginx (рекомендуемо при наличии VPS под Marzban)

    1. Создать файл `/var/www/import.bbtb.app/apple-app-site-association` с содержимым выше:
       ```bash
       sudo mkdir -p /var/www/import.bbtb.app
       sudo nano /var/www/import.bbtb.app/apple-app-site-association
       # вставить JSON, сохранить
       sudo chmod 644 /var/www/import.bbtb.app/apple-app-site-association
       ```

    2. Создать `/etc/nginx/sites-available/import.bbtb.app`:
       ```nginx
       server {
           listen 443 ssl http2;
           server_name import.bbtb.app;

           ssl_certificate /etc/letsencrypt/live/import.bbtb.app/fullchain.pem;
           ssl_certificate_key /etc/letsencrypt/live/import.bbtb.app/privkey.pem;
           ssl_protocols TLSv1.2 TLSv1.3;

           location = /.well-known/apple-app-site-association {
               default_type application/json;
               alias /var/www/import.bbtb.app/apple-app-site-association;
               add_header Cache-Control "public, max-age=3600";
           }

           location / {
               return 404;
           }
       }

       server {
           listen 80;
           server_name import.bbtb.app;
           return 301 https://$host$request_uri;
       }
       ```
       **NB:** Apple CDN does NOT follow 301/302 для AASA — но redirect 80→443 для нормального browser traffic ок; AASA fetched напрямую через 443.

    3. Включить site + получить cert:
       ```bash
       sudo ln -s /etc/nginx/sites-available/import.bbtb.app /etc/nginx/sites-enabled/
       sudo nginx -t
       sudo certbot --nginx -d import.bbtb.app
       sudo systemctl reload nginx
       ```

    ## Option B — Cloudflare Pages (zero-config alternative)

    1. Создать новый репозиторий `bbtb-aasa` (приватный OK) с структурой:
       ```
       bbtb-aasa/
       └── public/
           ├── _headers
           └── .well-known/
               └── apple-app-site-association
       ```

    2. `public/.well-known/apple-app-site-association` — содержимое JSON выше.

    3. `public/_headers`:
       ```
       /.well-known/apple-app-site-association
         Content-Type: application/json
         Cache-Control: public, max-age=3600
       ```

    4. Cloudflare Dashboard → Pages → Create project → Connect to Git → выбрать `bbtb-aasa`:
       - Build command: (empty)
       - Build output directory: `public`
       - Deploy.

    5. Pages settings → Custom domains → Add → `import.bbtb.app`:
       - Cloudflare auto-provisions SSL.
       - DNS должна быть в Cloudflare zone OR external DNS provider с CNAME на `bbtb-aasa.pages.dev`.

    ## Verification (после deploy — обязательные 4 проверки)

    ```bash
    # 1. Direct fetch — точный путь, без расширения, application/json.
    curl -I https://import.bbtb.app/.well-known/apple-app-site-association
    # Expected:
    #   HTTP/2 200
    #   content-type: application/json
    #   content-length: ~250

    # 2. JSON validity.
    curl -s https://import.bbtb.app/.well-known/apple-app-site-association | jq .
    # Должно: applinks.details[0].appIDs == [iOS, macOS], components[0] == { "/": "/import*" }

    # 3. No-redirect check.
    curl -sIL https://import.bbtb.app/.well-known/apple-app-site-association | grep -c "HTTP/"
    # Should be: 1 (один HTTP response — no 301/302 в chain).

    # 4. Apple CDN debug endpoint (CDN-processed form, after Apple fetched and cached).
    #    NB: at first deploy this may 404 — wait 5-10 min after a real device install/update
    #    that triggers Apple CDN fetch.
    curl "https://app-site-association.cdn-apple.com/a/v1/import.bbtb.app"
    # Returns Apple-processed AASA OR 404 если ещё не fetched.
    ```

    ## Common issues

    - **404 от Apple CDN после 30 минут:** Apple CDN fetches AASA при install/update приложения, не proactively. Установить Debug build на physical device → ждать 5 минут → retry curl. Or: add `?mode=developer` to entitlement (см. § «Debug bypass»).

    - **Content-Type: text/plain returned by nginx:** `default_type application/json` line missed OR overridden by `application/octet-stream`. Verify nginx site config + `nginx -t`.

    - **TLS errors при curl от device:** check Let's Encrypt cert chain `curl -v https://import.bbtb.app/...`; renew expired cert `sudo certbot renew`.

    - **Apple CDN кеш до 24h:** изменения AASA не видны на установленных устройствах ≤24h. Для UAT debug используем `?mode=developer` (см. § «Debug bypass»).

    3. **CRITICAL:** rollback это изменение BEFORE finalshining commit / TestFlight upload — `?mode=developer` в production entitlement не одобряется Apple Review.

    1. На iPhone — kill BBTB полностью (swipe up from app switcher).
    2. Открыть Telegram → послать самому себе тестовый URL `bbtb://import?url=https%3A%2F%2Fvpn.vergevsky.ru%2Fsub%2F<token>` (используй реальный Marzban subscription token).

    1. На iPhone — BBTB уже запущено (foreground OR background, but recently used).
    2. Switch to Telegram → tap to test URL again.
    3. **Expected:** BBTB foreground'ится, `ImportProgressOverlay` shows almost instantly (≤300ms perceived).

    1. На iPhone — open Safari → navigate to `https://import.bbtb.app/import?url=https%3A%2F%2Fvpn.vergevsky.ru%2Fsub%2F<token>`.
    2. **Expected:** Safari should NOT load the URL as a webpage. Instead, app banner appears OR app opens directly (depends on iOS version).
    3. Tap "Open in BBTB" banner OR app opens automatically.
    4. **PASS criteria:** BBTB receives URL via `.onContinueUserActivity` (verify в Console.app filter `DeepLinks` category — should see `notice` for ImportHandler.handle from importer category).
    5. **FAIL signs:** Safari loads URL as a 404 / blank page (AASA не подхватился — check `?mode=developer` flag в entitlement, ИЛИ wait 24h CDN cache + reinstall).

    **F3 (macOS) — Universal Link via Safari/Telegram-desktop:**
    1. На Mac — open Safari → navigate to same `https://import.bbtb.app/import?url=...`.
    2. **Expected:** BBTB.app activates (dock icon, window comes forward), import flow starts.
    3. **PASS criteria:** BBTB receives URL via `.onContinueUserActivity` (NOT `.onOpenURL` — Pitfall #1 verify). Check Console.app subsystem `app.bbtb.client` category `deep-links.importer` — should see ImportHandler.handle notice.
    4. **FAIL signs:** Safari loads URL as a webpage (Pitfall #1 — `.onContinueUserActivity` modifier missing OR macOS entitlement not configured).

    **F4 — Error UX (любая платформа):**
    1. Build a "broken" URL: `bbtb://import` (no `?url=` query parameter).
    2. Tap it (можно через Telegram OR Safari debug URL bar OR Shortcuts run-action).
    3. **Expected:** SwiftUI Alert appears с title "Не удалось открыть ссылку" + body "В ссылке отсутствует параметр «url». Попросите отправителя прислать ссылку заново." + OK button.
    4. Tap OK → alert dismisses, app stays open on MainScreen.
    5. **PASS criteria:** alert text matches L10n keys (alert.deep_link_error.title + deep_link.error.missing_parameter); ru locale verified.
