<objective>
Третий вертикальный слайс — UI server-list (D-08, D-10, D-11, UX-04 partial). После выполнения user может тапнуть по server-line на главном экране, увидеть sheet с серверами, разбитыми по подпискам, увидеть latency badges (заполняются прогрессивно), выбрать сервер или Auto, sheet закрывается с обновлением MainScreenViewModel.selectedServerID. Reconnect-on-active НЕ в этом плане (Plan 05).

Phase Goal (user story из ROADMAP):

**As a** пользователь BBTB, **I want to** видеть свои серверы сгруппированными по подпискам, переключаться между ними и обновлять список одним жестом, **so that** я мог подключаться к лучшему доступному серверу без ручной настройки.

Покрытие в этом плане: «видеть серверы сгруппированными» — yes; «переключаться» — selection пишется в state (фактический reconnect в Plan 05); «pull-to-refresh» — UI hook есть, fetch+refresh logic в Plan 04.

Purpose: UI делает foundation видимым для пользователя. Без UI Plans 01/02 — невидимая инфраструктура. Plans 04/05 расширят этот UI поведением (refresh, delete, reconnect).
